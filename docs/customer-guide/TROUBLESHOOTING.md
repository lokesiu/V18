# 故障排查 — V18 / 明证台

> 📌 **遇到问题先看这里**。90% 的问题都能在这里找到答案。

## 启动问题

### 应用无法启动（双击图标无反应）

**症状**: 双击 `launch_app.py` 或 `run_gui.bat` 后没反应。

**排查步骤**:
1. 检查 Python 是否安装（仅 `launch_app.py` 方式需要）
   - 打开命令提示符 → `python --version`
   - 应显示 Python 3.11 或 3.12
2. 检查依赖是否安装
   - `pip install -r requirements.txt`
3. 查看错误日志：直接用命令运行 `python launch_app.py`，看 stderr 输出
4. 重装 PySide6：`pip install --upgrade PySide6`

### 应用启动后闪退

**症状**: 启动后立即退出，无错误提示。

**排查**:
- 检查 Windows 事件查看器 → 应用日志
- 尝试以管理员身份运行
- 检查杀毒软件是否拦截（白名单 V18 目录）

### License 错误

参见 [ACTIVATION.md](ACTIVATION.md) 的故障排查章节。

## 运行问题

### 流水线跑到某阶段卡住不前进

**症状**: 进度条停在某个阶段超过 10 分钟。

**排查**:
1. 看 V18 主界面 → 进度详情 → 当前阶段名称
2. 如果是"AI 调用"阶段（DeepSeek / MiMo），检查：
   - 网络连接
   - DeepSeek / MiMo API Key 是否配置
   - API 服务商状态（去官网看是否有公告）
3. 如果是"渲染"阶段，检查：
   - 输出目录（`outputs/`）是否有写权限
   - 磁盘空间（建议 ≥ 5 GB 可用）
4. 重启 V18，会从断点继续

### AI 输出明显错误 / 跑题

**症状**: 流水线跑完，但生成的文书内容明显跑题或胡说八道。

**可能原因**:
- 上传材料质量差（图片模糊、PDF 是扫描件）
- 案件类型选错（选"交通事故"但材料是借贷纠纷）
- AI 模型临时降级

**解决**:
- 重新上传更高质量的材料
- 重新选择正确的案件类型
- 在设置里切换 AI 模型（如果有）
- 联系 V18 团队反馈（AI 输出应该有日志可查）

### 生成的 DOCX 文件打不开

**症状**: 输出的 DOCX 在 Word 里打开报错。

**排查**:
1. 检查文件大小（< 1 KB 通常是空文件）
2. 看 V18 主界面的质量门禁报告（应该有错误标红）
3. 看错误日志：`outputs/crash_logs/v18.log`
4. 重新跑流水线（如果是非确定性错误）

## 性能问题

### 应用运行慢

**症状**: UI 卡顿、流水线时间长。

**排查**:
- 检查电脑配置：V18 建议 ≥ 8 GB RAM, ≥ 4 核 CPU
- 关闭其他大型应用
- 检查是否有杀毒软件实时扫描 V18 目录（白名单）
- 检查 `outputs/` 目录是否累积太多旧文件（清理 > 30 天的）

### AI 调用超时

**症状**: AI 阶段提示"超时"。

**排查**:
- 检查网络（特别是用公司代理时）
- 尝试切换 AI 服务商（在设置里）
- 联系 V18 团队（可能是服务商问题）

## 错误消息

### "AI 子处理者不可用"

DeepSeek / MiMo API 不可达。检查：
- 网络连接
- API Key 是否正确
- 服务商状态

### "PipelineContext 校验失败"

数据流转中某字段格式错误。这是 V18 内部 bug，请：
- 把错误日志发给 V18 团队
- 附上案件材料类型（不要附材料本身！）

### "License 已过期 / 已失效"

参见 [ACTIVATION.md](ACTIVATION.md)。

### "内存不足"

V18 处理大文件需要较多内存。解决：
- 关闭其他应用
- 把大文件拆分成多个小文件
- 增加电脑内存（如果是长期使用）

## 数据问题

### 找不到之前的案件

**症状**: 之前跑过的案件在列表里看不到。

**可能原因**:
- 案件存储目录变更（V18 升级时可能改路径）
- 数据库文件损坏
- 误删

**排查**:
- 检查 `outputs/cases/` 目录（默认存储位置）
- 检查 V18 设置 → 数据存储位置
- 如果找不到，V18 团队可从 crash_log 恢复（需要事先启用）

### 生成的数据丢失

V18 默认不主动备份。**建议**:
- 定期备份 `outputs/` 目录
- 案件完成后立即下载交付物包
- 重要案件单独保存到外部存储

## AI 行为问题

### AI 输出包含幻觉 / 不实陈述

**症状**: 生成的文书包含材料中没有的内容。

**这是 AI 的固有限制**。V18 团队**强烈建议**：
- 律师**必须**专业审核每一份 AI 生成的文书
- 核对每个事实是否与原始材料一致
- 核对每个法条引用是否准确

V18 团队**不对**AI 输出错误导致的专业责任负责（详见 [Terms of Service](../legal/TERMS_OF_SERVICE.md)）。

### AI 输出包含偏见 / 不当内容

**症状**: 生成的文书包含歧视性、攻击性、违反职业伦理的内容。

立即停止使用并联系 V18 团队。V18 已实施反偏见 prompt，但 AI 模型仍可能出错。律所发现此类问题应：
- 截图保留证据
- 立即联系 V18 团队
- 必要时联系 AI 服务商（DeepSeek / MiMo）
